Why use Task Runner?

Web programming requires coordinating a lot of parallel operations. Let's call these tasks.

JavaScript applications have traditionally managed tasks by chaining together events or callbacks (often packaged as Promises). This works when the number of tasks is small but what about larger applications? Handling errors and interruptions can get very complicated when you have to track application state. The code responsible for managing this is often larger and more complex than the meat of your application.

Task Runner can help you out with this.

  • First, it introduces Task. Task is an interface that reprsents any kind of work your application may need to do. It's kind of like a Promise.
  • Next it provides several building block tasks that your application can use. (We'll learn more about these in a bit.) It also it provives an abstract implementation of Task that you can extend to create your own custom tasks.
  • It also provides ways to compose and organize tasks. If one task depends on another, Task Runner ensures that they're run in the correct order.
  • Lastly it gives you the ability to interrupt and resume your tasks. If an error happens, Task Runner helps cleanup and restart where things went wrong. This means you get to focus on the good stuff - your own application.

Task Lifecycle

Tasks have 5 basic states. The diagram below shows these states and how they are related.

To implement your own Task all you need to do is extend tr.Abstract and override the following methods:

  • runImpl: Called when a task is run.
  • interruptImpl: Called when a task is interrupted. This method should pause anything that was started when it ran.
  • resetImpl: Called when a task is reset. This method should restore the task to its initial state so that it can be rerun.

Task Runner Compatibility

This library is built with the Closure Compiler. It uses a subset of the Closure Library runtime (included with the distro) so that it can be used with any JavaScript framework- Angular, React, jQuery, etc.

If your project uses the full Closure library, you should use the Task Runner source and let Closure compile it along with the rest of your application. Otherwise you can use the built, compact version of Task Runner.
