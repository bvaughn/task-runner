Provides a mechanism for creating tasks via composition rather than inheritance.

This task-type decorates an Object (decorated) that defines a 'run' method. This run method should accept 2 parameters:

  1. complete: A callback to be invoked upon successful completion of the task. This callback accepts a parameter: data.
  2. error: A callback to be invoked upon task failure. This callback accepts 2 parameters: data and error-message.
  3. task: A reference to the decorator task.

The decorated Object may also implement 'interrupt' and 'reset' methods, although they are not required.

Constructor tr.Decorator(decorated, name)

Instance Methods

getDecorated()

Returns the decorated object.
