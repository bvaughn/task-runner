Runs a series of tasks and chooses the highest priority resolution based on their outcome.

This type of task can be used to implement things like application routers, where one or more dependencies are required. It provides an easy way to specify fallback behavior in the event that some of the dependencies fail.

Once the task completes, the chosen resolution will be accessible via the getResolution() method.

Constructor (name)

Instance Methods

addResolution()

Add a resolution (a task) and its prerequisite blocking tasks. Resolutions should be added in the order of highest-to-lowest priority.

getChosenResolution()

Returns the highest priority resolution that was able to be selected based on the outcome of the blocking tasks. This method should only be called once the resolver has finished running.
