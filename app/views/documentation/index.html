Abstract implementation of Task. Extend this task to create your own task with only a few lines of code.
Provides a mechanism for creating tasks via composition rather than inheritance.

Tasks used to perform common operations

These are a few built-in tasks for doing things that are common in web applications.

Waits for an event-dispatching target to trigger a specific type of event.
Creates Tasks from Promises and Promises from Tasks.
Animation-frame-based task for tweening properties.
Creates an XHR request and completes upon successful response from the server.

Tasks used to group other tasks

This type of task lets you treat a collection of tasks as a single task.

Lightweight interface to create a dependency graph task.
Executes a set of Tasks either in parallel or one after another.
Executes of a set of Tasks with dependencies on each other in the correct order.
Observes (but does not execute) a collection of Tasks. These tasks do not have to be related in any way.
Runs a series of tasks and chooses the highest priority resolution based on their outcome.

Tasks that decorate other tasks in order to change behavior

Use this type of task when an important decision needs to be deferred.
Decorates a task and re-dispatches errors as successful completions.
No-op task primarily useful for unit testing of default, op-op values (that can be replaced with meaningful tasks to customize behavior).
Decorator for tasks that should be retried on error. This can be useful for tasks that might fail intermittently due to things like a temporary loss of Internet connectivity.
Runs a series of tasks until one of them successfully completes.
Decorates a Task and enforces a timeout. If the decorated task has not completed within the speci fied time limits, the decorator errors.
Sleeps for an amount of time to pass before completing.
